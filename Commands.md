# Commands

---

# Database Migration

Every migration command requires the `--db <name>` flag to select which saved
session to run against. There is no implicit "active" session.

## `ez-mig up [count]`

Apply pending migrations. With no argument, apply all pending migrations to the
latest available version. With `count`, apply only the next `count` migrations.

```bash
ez-mig up --db <name>
ez-mig up <count> --db <name>
```

**Example**

```bash
ez-mig up --db local
ez-mig up 2 --db local
```

---

## `ez-mig down [count]`

Roll back applied migrations. With no argument, roll back a single migration.
With `count`, roll back that many migrations.

```bash
ez-mig down --db <name>
ez-mig down <count> --db <name>
```

**Example**

```bash
ez-mig down --db local
ez-mig down 3 --db local
```

---

## `ez-mig goto <version>`

Apply or roll back migrations until the specified migration version is reached.

```bash
ez-mig goto <version> --db <name>
```

**Example**

```bash
ez-mig goto 20260628153045 --db local
```

---

## `ez-mig force <version>`

Force the migration version without executing any migration scripts.

> **Warning**
>
> This command should only be used to recover from an inconsistent migration state.

```bash
ez-mig force <version> --db <name>
```

**Example**

```bash
ez-mig force 20260628153045 --db local
```

---

## `ez-mig version`

Display the current migration version.

```bash
ez-mig version --db <name>
```

**Example**

```bash
ez-mig version --db local
```

# Session Management

A session stores:

- Database connection URI
- Migration directory

Migration commands select a session by name via the `--db <name>` flag.

---

## `ez-mig config add <name>`

Create a new migration session.

```bash
ez-mig config add <name> --uri <database-uri> --path <migration-directory>
```

### Example

```bash
ez-mig config add local \
    --uri "postgres://user:password@localhost:5432/app?sslmode=disable" \
    --path "./migrations"
```

---

## `ez-mig config list`

List all saved sessions.

```bash
ez-mig config list
```

---

## `ez-mig config update <name>`

Update an existing session.

Only the flags provided are updated. Any omitted values remain unchanged.

```bash
ez-mig config update <name> [--uri <database-uri>] [--path <migration-directory>]
```

### Example

```bash
ez-mig config update local --path "./db/migrations"
```

---

## `ez-mig config remove <name>`

Delete a saved session.

```bash
ez-mig config remove <name>
```

---

# Typical Workflow

Create a session:

```bash
ez-mig config add local \
    --uri "postgres://user:password@localhost:5432/app?sslmode=disable" \
    --path "./migrations"
```

Apply all pending migrations:

```bash
ez-mig up --db local
```

Check the current migration version:

```bash
ez-mig version --db local
```

Migrate to a specific version:

```bash
ez-mig goto 20260628153045 --db local
```

Force a migration version (recovery only):

```bash
ez-mig force 20260628153045 --db local
```

Roll back the most recent migration:

```bash
ez-mig down --db local
```

---

# Notes

- `ez-mig` delegates all migration execution to **golang-migrate**.
- v1 supports `postgres` and `mysql` connection URIs only.
- Migration versions are the same versions generated by `golang-migrate`.
- Migration files must follow the standard `golang-migrate` naming convention.
- The `force` command updates the recorded migration version without executing migration files.
